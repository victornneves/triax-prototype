## Spacing Scale

Declared values align with the existing token scale in `src/styles/tokens.css`. All values are multiples of 4.

| Token | Value | CSS Variable | Usage |
|-------|-------|-------------|-------|
| xs | 4px | `var(--spacing-xs)` (0.25rem) | Icon gaps, badge internal vertical padding |
| sm | 8px | `var(--spacing-sm)` (0.5rem) | Table cell padding, badge pill horizontal padding |
| md | 16px | `var(--spacing-md)` (1rem) | Column padding, card padding |
| lg | 24px | `var(--spacing-lg)` (1.5rem) | Section padding |
| xl | 32px | `var(--spacing-xl)` (2rem) | Layout gaps, page padding |
| 2xl | 48px | `var(--spacing-2xl)` (3rem) | Major section breaks |

Source: `src/styles/tokens.css` lines 69-76 (pre-existing token scale, not changed in this phase).

Exceptions:
- Priority pill badge uses `padding: var(--spacing-xs) var(--spacing-sm)` (4px 8px). Compact table row density is achieved through `font-size-xs` (12px) rather than vertical padding reduction — the `xs` token (4px) is the minimum permitted vertical padding per the spacing scale. This matches the existing `profile-table__priority-badge` pattern at `Profile.css` line 112.
- Table header cells use `padding: var(--spacing-sm) var(--spacing-md)` — existing pattern from `HistoryPage.css` line 81, unchanged.

## Color

All colors via `var(--*)` tokens only. No raw hex values permitted in new CSS.

| Role | Token | Value | Usage |
|------|-------|-------|-------|
| Dominant (60%) | `var(--color-surface)` | #ffffff (light) / `--color-gray-900` (dark) | Table row background, list column background |
| Secondary (30%) | `var(--color-surface-subtle)` / `var(--color-surface-muted)` | gray-50 / gray-100 | Row hover state, active row highlight |
| Accent (10%) | MTS clinical color tokens | see MTS palette below | Priority pill badge backgrounds only |
| Destructive | `var(--mts-red)` / `var(--color-danger)` | #dc3545 | None in this phase — list is read-only |

Source: `src/styles/tokens.css` lines 48-67 (MTS tokens), lines 105-143 (semantic layer).

### Focal Point

Primary visual anchor for the history list screen: the priority pill badge — the only use of color in the list row, drawing the clinician's eye to triage priority first. All other list row elements use neutral text tokens to preserve the pill's visual prominence.

### Accent Reserved For

Accent (MTS clinical colors) is reserved exclusively for **priority pill badges in list rows**. These colors are immutable clinical identifiers — they must not be used for any non-priority UI element.

| MTS Color | Token | Badge label | Text color |
|-----------|-------|-------------|------------|
| Red | `var(--mts-red)` #dc3545 | Vermelho | white (`#fff`) |
| Orange | `var(--mts-orange)` #fd7e14 | Laranja | white (`#fff`) |
| Yellow | `var(--mts-yellow)` #ffc107 | Amarelo | `var(--mts-yellow-text)` = #000000 |
| Green | `var(--mts-green)` #28a745 | Verde | white (`#fff`) |
| Blue | `var(--mts-blue)` #007bff | Azul | white (`#fff`) |

## Component Inventory

These are the visual elements introduced or modified by this phase.

### New: `.history-page__priority-pill`

A compact colored pill badge for inline use inside table rows. Stacks on top of the global `.priority-badge .priority-{color}` classes to override their large sizing without altering the color definitions.

```
display: inline-block
padding: var(--spacing-xs) var(--spacing-sm)   /* 4px 8px */
border-radius: var(--radius-sm)                /* 0.5rem */
font-size: var(--font-size-xs)                 /* 0.75rem */
font-weight: 600
text-transform: uppercase
letter-spacing: 0.04em
```

Compact density is achieved through `font-size-xs` (12px), not through sub-4px vertical padding. The `xs` token (4px) is the minimum permitted vertical padding.

Special case: `.history-page__priority-pill.priority-yellow` must explicitly declare `color: var(--mts-yellow-text)` to prevent the white `color` from `.priority-badge` winning via specificity.

Source: RESEARCH.md § Pattern 6; Profile.css lines 111-125 (reference for compact pattern).

### Modified: `HistoryPage` table columns

Replace the 3-column table (Data | Hora | ID) with 4 enriched columns:

| # | Header (PT-BR) | CSS class | Content | Alignment |
|---|---------------|-----------|---------|-----------|
| 1 | Data/Hora | `.history-page__session-date` | `item.created_at` formatted via `toLocaleString('pt-BR', { timeZone: 'America/Sao_Paulo', day:'2-digit', month:'2-digit', year:'numeric', hour:'2-digit', minute:'2-digit' })` | left |
| 2 | Paciente | `.history-page__session-patient` | `item.patient_info?.name` or fallback `"Paciente anon."` | left |
| 3 | Prioridade | `.history-page__session-priority` | `<span className={priority.className + ' history-page__priority-pill'}>` | left |
| 4 | Duração | `.history-page__session-duration` | `formatDuration(item.stats?.duration_seconds)` | left |

Source: CONTEXT.md D-09; RESEARCH.md § Pattern 5.

### New: `src/utils/priority.js` (shared utility)

Not a visual component but part of the rendering contract. Exports `resolvePriority(triageResult)`. Always called as `resolvePriority(item.triage_result || {})` to guard against null `triage_result`. Returns `{ className, label }`.

Source: CONTEXT.md D-03; RESEARCH.md § Pattern 1.

## Interaction Contract

### Row selection

- Each table row is clickable via `onClick`.
- Active row highlighted with `.history-page__session-row--active` (background: `var(--color-surface-muted)`).
- Active row comparison uses `selectedSession?.session_id === item.session_id` — no more S3 key parsing.
- `handleSelectSession` calls `GET /history/{session_id}` (not `GET /history?key=...`).

Source: CONTEXT.md D-10, D-11; RESEARCH.md § Pattern 3, 4.

### Loading state

Claude's discretion (not locked by upstream decisions). Use the existing `.history-page__detail-loading` spinner pattern already in `HistoryPage.css` line 130 — a centered `<p>Carregando detalhes...</p>` for the detail column while a row fetch is in-flight. This is consistent and requires no new component.

### Sort order

Claude's discretion (not locked by upstream decisions). Sort sessions newest-first at `setSessions()` time: `(a, b) => new Date(b.created_at) - new Date(a.created_at)`. Mirrors Profile.jsx behavior. Apply in `loadSessions` after `response.json()`.

Source: RESEARCH.md § Pitfall 4.
