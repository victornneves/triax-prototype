**3. Add `useEffect` keydown listener** (after the `triggerShortcutFeedback` helper):
```jsx
useEffect(() => {
    const handleKeyDown = (e) => {
        // Suppress when focus is in form fields (D-08)
        const tag = document.activeElement?.tagName;
        if (tag === 'INPUT' || tag === 'TEXTAREA' || tag === 'SELECT') return;

        // Y/N shortcuts — only when yes/no question is pending (D-11)
        if (currentNode?.yesNo && !loading) {
            if (e.key === 'y' || e.key === 'Y') {
                e.preventDefault();
                triggerShortcutFeedback('sim');
                handleSendMessage('Sim');
                return;
            }
            if (e.key === 'n' || e.key === 'N') {
                e.preventDefault();
                triggerShortcutFeedback('nao');
                handleSendMessage('Nao');
                return;
            }
        }

        // R toggles recording (D-10)
        if ((e.key === 'r' || e.key === 'R') && !loading) {
            e.preventDefault();
            triggerShortcutFeedback('record');
            handleToggleRecording();
            return;
        }

        // Esc cancels active recording (D-09) — never suppressed
        if (e.key === 'Escape' && isRecording) {
            e.preventDefault();
            stopRecording();
            return;
        }
    };

    document.addEventListener('keydown', handleKeyDown);
    return () => document.removeEventListener('keydown', handleKeyDown);
}, [currentNode, loading, isRecording]);
```

Note: The dependency array includes `currentNode`, `loading`, and `isRecording` to avoid stale closures (per RESEARCH.md Pattern 5 / Pitfall 3). The handler will be re-registered on each state change of these values. `handleSendMessage`, `handleToggleRecording`, and `stopRecording` are stable references (defined inline or via useCallback) so they don't need to be in the deps array; however, if the build produces an ESLint warning about missing deps, add them.

```css
/* Shortcut hint text inside buttons: "(Y)", "(N)", "(R)", "(Esc)" */
.shortcut-hint {
  font-size: var(--font-size-sm);
  font-weight: 400;
  color: var(--color-text-muted);
  margin-left: var(--spacing-xs);
}

/* Pulse animation when keyboard shortcut fires (D-13) */
@keyframes shortcut-pulse {
  0%   { box-shadow: none; }
  40%  { box-shadow: 0 0 0 3px color-mix(in srgb, var(--color-primary) 30%, transparent); }
  100% { box-shadow: none; }
}

.shortcut-active {
  animation: shortcut-pulse 150ms ease-out forwards;
}
```

**Key decisions:**
- `.shortcut-hint` uses `--font-size-sm` (0.85rem / ~13.6px), weight 400, color `--color-text-muted` per UI-SPEC typography section. This provides visual differentiation from the button label's semibold weight without introducing a fifth font size.
- The pulse animation uses `box-shadow` with `color-mix()` to create a 30% opacity teal ring. This is cleaner than modifying `background-color` which would interfere with the button's existing background in both light and dark modes.
- 150ms duration matches D-13 specification.
- `forwards` fill mode keeps the final state (no box-shadow) so the button doesn't flash back.
