## Spacing Scale

Declared values from `tokens.css` `:root` (source: tokens.css):

| Token | CSS Variable | Value | Usage |
|-------|-------------|-------|-------|
| xs | `--spacing-xs` | 4px (0.25rem) | Icon gaps, inline padding |
| sm | `--spacing-sm` | 8px (0.5rem) | Compact element spacing |
| md | `--spacing-md` | 16px (1rem) | Default element spacing, button padding |
| lg | `--spacing-lg` | 24px (1.5rem) | Section padding, header padding |
| xl | `--spacing-xl` | 32px (2rem) | Layout gaps, form section padding |
| 2xl | `--spacing-2xl` | 48px (3rem) | Major section breaks |

**Phase 8 exceptions:**

- Dark mode toggle button: 36px x 36px minimum touch target (icon-only button in header — use `min-width: 36px; min-height: 36px; padding: var(--spacing-sm)`)
- Waveform canvas: 120px height x 100% width of the recording panel — not a spacing token; defined as a fixed component dimension
- Recording panel: `min-height: 80px` to house waveform + timer side by side (not a spacing token)
- Shortcut hint labels on yes/no buttons: additional `4px` horizontal padding beyond button base padding — handled inline as `gap: var(--spacing-xs)` between label and shortcut hint

## Typography

Declared values from `tokens.css` `:root` `--font-size-*` scale (source: tokens.css):

| Role | CSS Variable | px Equivalent | Weight | Line Height | Usage |
|------|-------------|---------------|--------|-------------|-------|
| Body | `--font-size-md` | 16px (1rem) | 400 (regular) | 1.5 | Chat messages, transcript preview, form text |
| Label / Caption | `--font-size-sm` | ~13.6px (0.85rem) | 400 (regular) | 1.4 | Shortcut hints "(Y)" "(N)" "(R)", timer, nav links, button labels |
| Heading | `--font-size-xl` | 20px (1.25rem) | 600 (semibold) | 1.2 | Section headings, modal titles |
| Brand / Display | `--font-size-2xl` | 24px (1.5rem) | 600 (semibold) | 1.2 | Header brand "Triax" |

**Phase 8 specifics:**

- Shortcut key hints (`(Y)`, `(N)`, `(R)`) use `--font-size-sm` (~13.6px / 0.85rem), weight 400 (regular), color `--color-text-muted`. Visual differentiation from the button label is achieved via muted color and regular weight — not a separate smaller size.
- Elapsed timer: `--font-size-sm` (0.85rem), weight 600 (semibold), monospace rendering via `font-variant-numeric: tabular-nums` to prevent digit-width jitter as seconds advance.
- Live transcript preview text: `--font-size-md` (1rem), weight 400, line-height 1.5 — matches body text to look like normal input content.

**Weights in use:** 400 (regular) and 600 (semibold) only — no other weights declared or allowed.

**Active size count: 4** — sm, md, xl, 2xl. No fifth size. `--font-size-xs` is not used in Phase 8.

## Color

All values from `tokens.css` semantic layer on `[data-app-theme]` (source: tokens.css):

| Role | Light Mode Value | Dark Mode Value | Usage |
|------|-----------------|-----------------|-------|
| Dominant (60%) — surface | `--color-surface: #ffffff` | `--color-surface: #1c1917` (gray-900) | Page background, chat column, recording panel |
| Secondary (30%) — subtle/muted | `--color-surface-subtle: #fafaf9` | `--color-surface-subtle: #292524` (gray-800) | Recording panel background, sensor column |
| Accent (10%) — primary | `--color-primary: #14b8a6` (teal-500) | `--color-primary: #2dd4bf` (teal-400) | Dark mode toggle active state indicator, waveform canvas line color, shortcut pulse highlight |
| Destructive | `--color-error-text: #991b1b` | `--color-error-text: #fca5a5` | Cancel recording button (danger variant) |

**Accent reserved for:**
1. Dark mode toggle button — active state ring or icon fill when in dark mode
2. Waveform oscilloscope canvas — line stroke color (`--color-primary`)
3. Shortcut pulse highlight — `.shortcut-active` 150ms flash uses `--color-primary` as a background tint at 20% opacity

**MTS clinical colors (`--mts-*`):** Immutable across both modes. No Phase 8 changes. Never aliased or softened. (source: STATE.md accumulated decisions)

**Dark mode toggle — new CSS class design:**
- Sun/moon icon button: background `transparent`, color `--color-text-secondary` in both modes
- Hover: background `--color-nav-bg-hover`, color `--color-text-primary`
- No separate "active" state coloring — the icon itself (sun vs. moon) communicates current mode
- Transition: `background-color 200ms ease, color 200ms ease` (matches `--transition-fast` at 150ms, extended to 200ms for theme crossfade per D-03)

**Dark mode crossfade on `[data-app-theme]`:**
```css
[data-app-theme] {
  transition: background-color 200ms ease,
              color 200ms ease,
              border-color 200ms ease;
}
```
Applied to `[data-app-theme]` root selector. Individual child elements inherit or override as needed. 200ms `ease` curve — not linear, not spring. (source: CONTEXT.md D-03, Claude's Discretion)

## Component Inventory

New components and modifications required for Phase 8:

### New: `ThemeContext.jsx`
**Path:** `src/contexts/ThemeContext.jsx`
**Pattern:** Mirrors `src/contexts/UserContext.jsx` exactly. Exports `ThemeProvider` and `useTheme`.
**API surface:**
- `theme`: `'light' | 'dark'`
- `toggleTheme()`: flips theme, writes `localStorage.triax-theme`, updates `data-app-theme` on `.app-container`
**Init logic:** `localStorage.triax-theme` → fallback to `window.matchMedia('(prefers-color-scheme: dark)').matches` → fallback to `'light'`
(source: CONTEXT.md D-04, D-07; RESEARCH.md Pattern 1)

### New: Dark Mode Toggle Button (in `Header.jsx`)
**Placement:** `.header-right` section, leftmost element before the profile link
**Markup:** `<button class="header-theme-toggle" aria-label="Alternar modo escuro">` containing sun (☀) or moon (🌙) character
**Dimensions:** 36px x 36px touch target, `border-radius: var(--radius-md)`, `border: none`, `background: transparent`
**Tooltip:** `title="Modo escuro" / title="Modo claro"` (native browser tooltip — no custom Tooltip component needed for this simple toggle)
(source: CONTEXT.md D-01, D-02)

### New: Recording Panel (replaces mic button section during `isRecording === true`)
**Path:** New CSS block in `src/components/ProtocolTriage.css`
**Layout (D-17):** Flex row: transcript preview (flex: 1, primary) | waveform + timer column (fixed 140px, secondary)
**Transcript preview:** Full-height flex child, padding `var(--spacing-md)`, background `--color-surface-subtle`, border-radius `var(--radius-md)`, border `1px solid --color-border`, overflow-y auto
**Waveform canvas:** 120px height, 140px width, background `transparent`, line color `--color-primary`, line width `2px`
**Elapsed timer:** Below the waveform, `--font-size-sm`, weight 600, `font-variant-numeric: tabular-nums`, color `--color-text-secondary`, text-align center
**Stop button:** Inside the recording panel, `btn btn--danger btn--sm`, label "Parar (Esc)" with shortcut hint
**Transition into recording panel:** CSS `opacity` + `max-height` transition from the input bar, 200ms ease (same crossfade curve as dark mode for consistency)
(source: CONTEXT.md D-15, D-16, D-17, D-18, D-19)

### Modified: Yes/No Buttons (in `ProtocolTriage.jsx`)
**Change:** Label text becomes "Sim (Y)" and "Não (N)" — shortcut hint rendered as `<span class="shortcut-hint">(Y)</span>` inside button, not inline with main label text
**Shortcut hint span:** `--font-size-sm`, color `--color-text-muted`, weight 400, `margin-left: var(--spacing-xs)`. Muted color and regular weight (vs. button label's semibold) provide visual differentiation without introducing a fifth font size.
**Pulse class:** `.shortcut-active` — added via JS for 150ms, then removed. CSS: `animation: shortcut-pulse 150ms ease-out forwards`
```css
@keyframes shortcut-pulse {
  0%   { background-color: inherit; }
  40%  { background-color: color-mix(in srgb, var(--color-primary) 20%, transparent); }
  100% { background-color: inherit; }
}
```
(source: CONTEXT.md D-12, D-13)

### Modified: Mic Button (in `ProtocolTriage.jsx`)
**Change:** Label becomes "Gravar (R)" with shortcut hint `<span class="shortcut-hint">(R)</span>`
**Shortcut hint:** Same `.shortcut-hint` class as yes/no buttons
**During recording:** Mic button section is hidden — replaced by recording panel. Toggle via `isRecording` conditional render.
(source: CONTEXT.md D-10, D-15)

## Interaction Contracts

### Dark Mode Toggle (DSGN-03)

| State | Visual |
|-------|--------|
| Light mode active | Moon icon (🌙) shown, `aria-label="Ativar modo escuro"` |
| Dark mode active | Sun icon (☀) shown, `aria-label="Ativar modo claro"` |
| Hover | `--color-nav-bg-hover` background, `--color-text-primary` icon color |
| Focus-visible | `outline: 2px solid --color-primary; outline-offset: 2px` (matches `.btn:focus-visible` pattern) |
| Transition | 200ms `ease` crossfade on `background-color`, `color`, `border-color` applied to `[data-app-theme]` |

**Persistence:** `localStorage.triax-theme` key, value `'light'` or `'dark'`. Written on every toggle. Read on mount by ThemeContext and by FOUC script.

**FOUC script update (`index.html`):** Existing script reads `localStorage.triax-theme`. Extend to: if no stored value, check `window.matchMedia('(prefers-color-scheme: dark)').matches` and apply `'dark'` to `data-app-theme` before React hydrates. (source: CONTEXT.md D-05)

### Keyboard Shortcuts (INTR-02)

| Key | Action | Condition | Suppressed When |
|-----|--------|-----------|-----------------|
| `Y` | Fire `handleSendMessage('Sim')` | `currentNode?.yesNo === true` | Focus in `input`, `textarea`, `select` |
| `N` | Fire `handleSendMessage('Não')` | `currentNode?.yesNo === true` | Focus in `input`, `textarea`, `select` |
| `R` | Toggle voice recording (`handleToggleRecording`) | ProtocolTriage route active | Focus in `input`, `textarea`, `select` |
| `Escape` | Cancel/stop active recording (`stop()`) | `isRecording === true` | Never suppressed (standard Esc convention) |

**Listener attachment:** Single `keydown` listener in `useEffect` inside `ProtocolTriage.jsx`. Cleaned up on unmount (`return () => document.removeEventListener('keydown', handler)`).

**Input guard:** `if (['INPUT', 'TEXTAREA', 'SELECT'].includes(e.target.tagName)) return;` — checked before any shortcut action.

**Visual feedback:** On shortcut fire, add `.shortcut-active` CSS class to the corresponding button element for exactly 150ms via `setTimeout`. The class triggers a `shortcut-pulse` keyframe animation (teal tint flash). (source: CONTEXT.md D-13)

### Voice Recording UI (INTR-03)

| State | What Clinician Sees |
|-------|---------------------|
| Not recording | Normal text input bar with mic button labeled "Gravar (R)" |
| Recording (isRecording = true) | Text input bar replaced by: transcript preview (left, primary) + waveform canvas + timer (right, secondary) + "Parar (Esc)" button |
| Partial transcript arriving | `partialTranscript` shown in transcript preview with italic styling; `finalTranscript` shown in regular weight above it |
| Recording stopped | Recording panel collapses back to text input, input auto-populated with `finalTranscript + ' ' + partialTranscript`. Clinician reviews, then presses Enter or "Enviar" to send. No auto-send. |

**Waveform rendering (Claude's Discretion):**
- `requestAnimationFrame` loop while `isRecording === true`; loop cancels when recording stops
- Audio data: `onAudioData` callback added to `useTranscribe` hook, fires during `processor.onaudioprocess`, receives `Float32Array` from `getChannelData(0)`
- Canvas draw: clear canvas, compute per-sample amplitude, draw oscilloscope line from left to right across canvas width
- Line style: `strokeStyle = var(--color-primary)` computed via `getComputedStyle`, `lineWidth = 2`, `lineCap = 'round'`
- When not recording: canvas shows flat line at vertical midpoint (not blank)

**Elapsed timer:**
- Starts at `0` seconds when recording begins (`isRecording` transitions to `true`)
- `setInterval(1000ms)` increments a `recordingSeconds` local state
- Cleared in the `useEffect` cleanup when `isRecording` becomes `false`
- Display format: `M:SS` (e.g. `0:04`, `1:32`) — not `00:04`
- Font: `--font-size-sm`, weight 600, `font-variant-numeric: tabular-nums`

**Transcript preview partial/final distinction:**
- `finalTranscript`: black (`--color-text-primary`), weight 400
- `partialTranscript`: color `--color-text-muted`, italic (`font-style: italic`) — visually communicates "still transcribing"
- Both rendered in the same scrollable preview area, final above partial

## Implementation Notes for Executor

These notes translate design decisions into concrete implementation constraints:

1. **ThemeContext file location:** `src/contexts/ThemeContext.jsx` (not co-located with Header). Matches `UserContext.jsx` placement. (source: RESEARCH.md Architecture Patterns)

2. **`data-app-theme` attribute:** Currently hardcoded `"light"` on the `.app-container` div in `App.jsx` line 42. Must become `data-app-theme={theme}` where `theme` comes from `useTheme()`. Do NOT use a ref approach — JSX prop is cleaner. (source: RESEARCH.md Pattern 1)

3. **ThemeProvider mount position in `App.jsx`:** Wrap inside `Authenticator` (same level as `ToastProvider` and `UserProvider`) so toggle is only rendered when logged in. (source: RESEARCH.md Architecture Patterns)

4. **`useTranscribe.js` modification scope:** Add `onAudioData` prop (callback) to the hook's parameter. Inside `processor.onaudioprocess`, if `onAudioData` is provided, call `onAudioData(inputData.getChannelData(0))`. Existing behavior is unchanged when callback is not passed. (source: CONTEXT.md D-21)

5. **`keydown` listener coexistence:** `ProtocolTriage.jsx` line 1155 has an existing Enter key handler on the text input. The new `keydown` listener is on `document`, not on the input. The input's own `onKeyDown` prop for Enter is unaffected. (source: CONTEXT.md D-14; canonical refs in CONTEXT.md)

6. **Atomic migration rule:** Each modified file gets a complete pass — no mixed inline styles and class-based styles in the same file. (source: STATE.md accumulated decisions)

7. **MTS colors during dark mode:** `--mts-*` tokens do not change. In `[data-app-theme="dark"]` there are no overrides for MTS tokens — by design. Clinical priority colors are mode-invariant. (source: tokens.css lines 145-181, STATE.md decisions)

8. **`.shortcut-hint` class typography:** Always `--font-size-sm`, weight 400, color `--color-text-muted`. Do not introduce `--font-size-xs` or any fifth font size. The type scale is exactly 4 sizes: sm, md, xl, 2xl.
