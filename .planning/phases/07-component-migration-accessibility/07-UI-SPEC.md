## Spacing Scale

All spacing uses tokens already defined in `src/styles/tokens.css`. No new spacing tokens are needed for Phase 7 — only the radius/shadow/transition tokens listed below are additions.

| Token | CSS Variable | Pixel Equivalent | Usage |
|-------|-------------|-----------------|-------|
| xs | `--spacing-xs` (0.25rem) | 4px | Icon gaps, asterisk indicator spacing, tooltip arrow offset |
| sm | `--spacing-sm` (0.5rem) | 8px | Compact element spacing, inline error message top margin, label-to-input gap |
| md | `--spacing-md` (1rem) | 16px | Default element spacing, form field gap, cell padding |
| lg | `--spacing-lg` (1.5rem) | 24px | Section padding, card padding, status bar horizontal padding |
| xl | `--spacing-xl` (2rem) | 32px | Layout gaps, page container padding, section breaks |
| 2xl | `--spacing-2xl` (3rem) | 48px | Major section breaks between form groups |

**Exceptions (source: CONTEXT.md):**
- Mobile vitals toggle button (pill shape, fixed bottom-right): minimum touch target 44px height — use `min-height: 44px` directly, not a spacing token.
- Focus ring `outline-offset`: 2px — intentionally sub-token, not from the spacing scale (A11Y-03 requirement).
- Status bar height: 32px fixed — sub-token value for a utility bar that must not reflow content.

**New tokens to add to `tokens.css` (CONTEXT.md D-05):**

| Token | Value | Usage |
|-------|-------|-------|
| `--radius-sm` | 0.25rem | Small badge corners, tooltip corners |
| `--radius-md` | 0.5rem | Card corners, form inputs, modals |
| `--radius-lg` | 0.75rem | Sensor panel drawer on mobile, pill button |
| `--shadow-sm` | `0 1px 2px rgba(0,0,0,0.06)` | Cards, form inputs on focus |
| `--shadow-md` | `0 4px 12px rgba(0,0,0,0.12)` | Sensor drawer overlay, tooltip |
| `--transition-fast` | `150ms ease` | All CSS transitions (collapse, tooltip fade, focus ring) |

## Typography

Sourced from `tokens.css` primitive layer. All type is in pt-BR. No new font scale additions for Phase 7.

| Role | CSS Variable | Pixel Equivalent | Weight | Line Height | Usage |
|------|-------------|-----------------|--------|-------------|-------|
| Body | `--font-size-md` (1rem) | 16px | 400 (regular) | 1.5 | All body text, form field values, table cells, tooltip content |
| Label | `--font-size-sm` (0.85rem) | ~14px | 600 (semibold) | 1.4 | Form labels, table headers, status bar text |
| Heading | `--font-size-xl` (1.25rem) | 20px | 600 (semibold) | 1.2 | Page titles, modal headings, section headings |
| Caption | `--font-size-xs` (0.75rem) | 12px | 400 (regular) | 1.4 | Inline validation error messages, compact vitals strip, tooltip meta |

**Weight constraint:** Exactly 2 weights used — 400 (regular) and 600 (semibold). No 500, no 700.

**MTS badge text exception:** MTS priority badge labels use weight 700 (bold) on colored backgrounds — this is a clinical readability requirement, not a design choice. Source: v1.1.0 App.css immutability constraint.

## Color

All values sourced from `src/styles/tokens.css`. No new color tokens are added in Phase 7 — only contrast compliance enforcement (D-20 to D-22).

### 60/30/10 Distribution

| Role | Token | Resolved Value | Usage |
|------|-------|---------------|-------|
| Dominant (60%) | `--color-surface` | `#ffffff` (light) / `--color-gray-900` (dark) | Page background, form backgrounds, modal backdrop content area |
| Secondary (30%) | `--color-surface-subtle` / `--color-surface-muted` | `#fafaf9` / `#f5f5f4` | Cards, sidebar panel, table row stripes, status bar background, sensor panel |
| Accent (10%) | `--color-primary` | `--color-teal-500` (#14b8a6) | See reserved-for list below |
| Destructive | `--color-feedback-error-text` | `#991b1b` | Inline validation errors, error border, error toast (already wired in Phase 6) |

**Accent (`--color-primary`) reserved exclusively for:**
1. Active/focus focus ring (`outline: 2px solid var(--color-primary)` on `:focus-visible`)
2. Primary action buttons (already wired in `Button.jsx` variant="primary")
3. Connection indicator "online" dot in StatusBar — use `--color-feedback-ok-text` (`#166534`) not accent; accent is teal, green dot must be green
4. Links that require distinction from body text

**Accent is NOT used for:** table headers, section dividers, form labels, tooltip triggers, or status bar background.

### Semantic color constraints (A11Y-01 compliance)

| Failing token / hex | Contrast on white | Replacement | Contrast |
|--------------------|-------------------|-------------|---------|
| `#adb5bd` (raw hex, various files) | 2.07:1 — FAIL | `--color-text-secondary` (`--color-gray-600`, #57534e) | 7.63:1 — PASS |
| `#868e96` (raw hex, various files) | 3.32:1 — FAIL | `--color-text-secondary` (`--color-gray-600`, #57534e) | 7.63:1 — PASS |
| `#999`, `#666`, `#555` (raw hex) | varies, many fail | `--color-text-secondary` or `--color-text-primary` depending on context | PASS |
| `--color-text-muted` (`--color-gray-400`, #a8a29e) | 2.52:1 — FAIL on body text | Only permitted for decorative content (placeholder, disabled state). Never on readable text. | n/a |

**Rule (D-22):** Every `.css` file created in Phase 7 uses semantic token references only. No raw hex values in new component CSS files.

### MTS clinical colors — immutable

`--mts-red`, `--mts-orange`, `--mts-yellow`, `--mts-green`, `--mts-blue` are byte-for-byte from v1.1.0. Never aliased, never softened. `--mts-yellow-text: #000000` companion token already exists (Phase 5). Patient safety constraint — out of scope for any Phase 7 modification.

### Connection status indicator colors (StatusBar, D-12)

| State | Visual | Token |
|-------|--------|-------|
| Online | Green dot (#12px circle) | `--color-feedback-ok-text` (#166534) as background, or inline `background-color: var(--color-feedback-ok-text)` |
| Offline | Red dot + "Sem conexão" text | `--color-feedback-error-text` (#991b1b) |

## Interaction Contracts

### Focus indicator (A11Y-03, D-17/D-18)

Applied globally in `src/index.css`. Replaces browser default.

```
:focus-visible {
  outline: 2px solid var(--color-primary);
  outline-offset: 2px;
}
```

- Applies to: all inputs, buttons, links, selects, textareas, and any element with `tabindex`
- WebKit default (`outline: 4px auto -webkit-focus-ring-color`) is removed
- Dark mode: `--color-primary` already has dark override (`--color-teal-400`) — no additional dark rule needed

### Sensor panel responsive behavior (LAYT-01, D-06 to D-09)

| Viewport | Behavior |
|----------|----------|
| Desktop (≥768px) | `<aside>` sidebar always visible alongside chat column. Two-column flex layout. |
| Mobile (<768px) | Collapsed by default. Pill-shaped toggle button fixed bottom-right (min-height 44px). Expanded = slide-up overlay using `transform: translateY(0)` transition. |

- Collapse/expand: CSS `transform: translateY` transition using `--transition-fast` (150ms). No JS animation library.
- Collapsed mobile state: compact vitals strip visible above toggle button — format: `SpO2: {val}% | FC: {val} | T: {val}°C` — single line, `--font-size-sm`, `--color-text-secondary`.
- State: `useState(false)` in `ProtocolTriage.jsx`. Boolean `isPanelOpen`.
- Overlay backdrop: `rgba(0,0,0,0.4)` semi-transparent. Click outside closes panel.

### Form validation (FORM-01, D-23 to D-26)

| State | Visual treatment |
|-------|-----------------|
| Default | `--color-border` border, no message |
| Error (on blur or submit) | `--color-feedback-error-text` border (2px solid), inline error message below field (`--font-size-xs`, `--color-feedback-error-text`), `aria-invalid="true"` on input, `aria-describedby` pointing to error `<span>` |
| Valid (after correcting error) | `--color-border` border restored, error message removed |

- Required field label marker: red asterisk (`*`) in `--color-feedback-error-text` immediately after label text, `aria-hidden="true"` on the asterisk span.
- Validation timing: on `blur` per field; on `submit` for all required fields simultaneously.
- No library — custom logic per field.

### Modal focus trap (A11Y-02, D-15)

- `TriageDetailsModal.jsx`: `role="dialog"`, `aria-modal="true"`, `aria-labelledby="{modal-title-id}"`.
- On open: focus moves to first focusable element inside modal.
- Tab cycles within modal; Shift+Tab reverses. Focus does not escape to page behind.
- On close: focus returns to the element that triggered the modal (store ref before open).
- Native DOM implementation (`useRef` + `keydown` handler) — no `focus-trap-react` library.

### Tooltip behavior (FORM-03, D-29 to D-31)

- Trigger: `ⓘ` Unicode character as `<button type="button">` inline after label text.
- Trigger size: `--font-size-sm`, `--color-text-muted`, `cursor: help`.
- Show on: hover OR `:focus-visible` on trigger button.
- Hide on: mouse leave AND blur.
- Position: above trigger by default; auto-flip below if viewport top edge is clipped.
- Content: `--font-size-xs`, `--color-text-primary` on `--color-surface` background, `--shadow-md`, `--radius-sm`.
- Max width: 240px.
- `role="tooltip"` on the tooltip element; `aria-describedby` on the trigger pointing to tooltip id.

### Input masking (FORM-04, D-32 to D-34)

| Field | Mask | Stored value (unformatted) |
|-------|------|---------------------------|
| Data de nascimento | `DD/MM/AAAA` | `YYYY-MM-DD` for API or raw digits |
| CPF | `000.000.000-00` | 11 digits only |
| Pressão arterial sistólica | `{0-9}{0-9}{0-9}` (max 3 digits) | raw number |
| Pressão arterial diastólica | `{0-9}{0-9}{0-9}` (max 3 digits) | raw number |

- Library: `react-imask` 7.6.1 — `IMaskInput` component.
- Use `onAccept` callback to update state with unmasked value.
- `unmask: true` on `IMaskInput` where API receives raw value.

### Age auto-calculation (FORM-02, D-27/D-28)

- Trigger: `onChange` on birth_date field when masked value is a complete valid date.
- Effect: `idade` field is set to calculated age in years (integer). Field remains editable.
- Algorithm: current year − birth year, adjusted for month/day (no date library).
- If birth_date is cleared or invalid: age field reverts to empty (clinician must fill manually).

### Global status bar (LAYT-02, D-10 to D-12)

- Component: `StatusBar.jsx` in `src/components/ui/`.
- Height: 32px fixed.
- Background: `--color-surface-muted`.
- Border-bottom: `1px solid var(--color-border)`.
- Visibility: conditional — only rendered inside `ProtocolTriage` route.
- Content layout (left to right): `[session ID truncated] · [protocol name] · [● online / ● Sem conexão]`
- Session ID: first 8 chars of the ID, monospace font optional, `--font-size-xs`.
- Protocol name: truncated at 24 chars with ellipsis if longer.
- Spacing between items: `·` separator character with `--spacing-sm` horizontal padding on each side.

## Semantic HTML Contract

Applies to all components migrated in Phase 7 (A11Y-02). Source: CONTEXT.md D-13 to D-16.

| Element type | Before (current) | After (required) |
|-------------|-----------------|-----------------|
| Form labels | `<div style={{...}}>` wrapping label text | `<label htmlFor="{input-id}">` with matching `id` on input |
| Header profile link | `<div onClick={...}>` | `<button type="button">` or `<Link>` |
| Modal wrapper | Plain `<div>` | `<div role="dialog" aria-modal="true" aria-labelledby="{title-id}">` |
| Modal title | Unsemantic heading | `<h2 id="{title-id}">` matched to `aria-labelledby` |
| Tables | No caption, no scope | `<caption>` element + `scope="col"` on all `<th>` header cells |
| Sensor panel | Unsemantic div | `<aside aria-label="Painel de sinais vitais">` |
| Status bar | — (new component) | `<div role="status" aria-live="polite">` |
| Tooltip | — (new component) | `role="tooltip"`, trigger uses `aria-describedby="{tooltip-id}"` |
