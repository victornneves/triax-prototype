**Accent reserved for (explicit list):**
1. Focus border on primary inputs (`:focus` state of `.patient-form__input`)
2. Focus border on metadata inputs (`:focus` state of `.patient-form__meta-input`)
3. CPF spinner arc border-top-color (`--color-primary`)
4. Submit button background (`--color-primary`) and hover (`--color-primary-hover`)

The accent color is NOT used for: labels, section dividers, metadata card backgrounds, or
the age display label. Those use gray tokens only.

**MTS clinical colors:** Not used in PatientForm. The form predates triage session — no
priority color should appear here. If PatientForm is extracted, confirm zero `--mts-*` usage.

**Primary visual anchor:** CPF input (top of form, first tab stop, draws clinician attention on render).

## Interaction Contract

### CPF Field (D-01, D-02, D-03, D-04, D-05)

| State | Visual | Behavior |
|-------|--------|----------|
| Empty / typing | Subtle gray background (`--color-surface-muted`), lighter border (`--color-border`) | IMask formats as `000.000.000-00` on each keystroke via `onAccept` |
| Focused | Teal border (`--color-primary`), 3px box-shadow teal 15% opacity | Standard focus ring — same as all primary inputs |
| Blur — invalid (< 11 digits or invalid CPF format) | Error border (`--color-feedback-error-text`, 2px), inline error text below | `handleCpfBlur` runs validation; no lookup |
| Blur — valid 11 digits (lookup active) | Spinner appears inside field at right (1em, border-top teal, `@keyframes btn-spin`) | `isLookingUp = true`; other fields remain editable; no skeleton/overlay |
| Lookup complete — match found | Spinner disappears; Name, Sex, Birth Date auto-fill | Auto-filled fields use `--color-surface-muted` background, remain editable (D-04) |
| Lookup complete — no match | Spinner disappears; all fields stay empty and editable | No toast, no message (D-05) |
| Lookup complete — error | Spinner disappears; toast via existing ToastProvider: "Erro ao buscar paciente. Tente novamente." | Follows existing toast error pattern (role="alert") |

### Age Display (D-10, D-11, D-12)

| State | Visual | Behavior |
|-------|--------|----------|
| No birth date entered | Shows "—" in muted text (`--color-text-muted`) | Static |
| Birth date partially entered | Shows "—" until full valid date | `calcAgeFromDDMMYYYY` returns null on partial date |
| Birth date complete | Shows "{N} anos" in secondary text (`--color-text-secondary`) | Computed on every `onAccept` callback from birth_date IMask field |

`aria-live="polite"` on the age display element ensures screen readers announce the computed age without interrupting the user.

### Metadata Section (D-13, D-14, D-15, D-16)

- Always visible; never hidden or collapsed (D-15)
- Section label "DADOS ADMINISTRATIVOS": `--font-size-xs`, weight 600, `--color-text-muted`, `text-transform: uppercase`, `letter-spacing: 0.05em`
- Grid layout: 3 columns on desktop (`repeat(3, 1fr)`), 2 columns on narrow (< 480px via media query) — Claude's discretion, locks here as 3-col desktop / 2-col mobile
- Metadata inputs use `.patient-form__meta-input` class: `--color-surface-muted` background, `--color-border` border (or `1px solid transparent` for borderless variant — use bordered for better contrast in dark mode), `--font-size-sm` text, `--color-text-secondary` color
- Top border-top divider: `1px solid var(--color-border)` + `--spacing-md` padding-top and margin-top

### Submit Button (D-09)

- `.patient-form__submit-wrapper`: `position: sticky; bottom: 0; background-color: var(--color-surface); padding-top: var(--spacing-md); box-shadow: 0 -2px 8px rgba(0,0,0,0.06); z-index: 1`
- The scroll container (`.patient-form-wrapper`) must have `overflow-y: auto` and `max-height: 100dvh` for sticky to activate. If current `align-items: center` on flex blocks scroll, add inner scroll wrapper.
- Submit button remains `.patient-form__submit` (full-width, teal, `--font-size-lg`, weight 600) — or replaced by `<Button variant="primary" loading={loading}>` from shared Button component to get spinner for free
- Disabled state: `opacity: 0.7; cursor: not-allowed` (existing pattern, keep)

### Tab Order (D-08)

DOM order is the tab order. JSX must be arranged:
1. CPF (top of form)
2. Name
3. Birth Date
4. Sex (select)
5. Metadata section (Patient Code, Ticket, Insurance, Visit ID, SAME)
6. Submit wrapper

No `tabIndex` attributes needed; DOM order is sufficient.

### Input Focus — All Primary Fields (D-17, D-18)

Refined `.patient-form__input`:
- Rest: `--color-surface-muted` background, `--color-border` border (lighter than existing `--color-border-strong`)
- Focus: `border-color: var(--color-primary)`, `box-shadow: 0 0 0 3px rgba(20, 184, 166, 0.15)`, `background-color: var(--color-surface)` (lighten on focus)
- Error: `border-color: var(--color-feedback-error-text)`, `border-width: 2px` (existing `.patient-form__input--error` — keep as-is)
- Transition: `border-color var(--transition-fast), box-shadow var(--transition-fast), background-color var(--transition-fast)`

### Validation — Inline Errors (FORM-05 success criterion 6)

- Error text rendered via `.patient-form__error` below the field — keep existing class
- Error border via `.patient-form__input--error` — keep existing class
- Validation trigger: `onBlur` for all fields (existing `handleBlur` pattern)
- CPF field: `handleCpfBlur` replaces `handleBlur` — runs validation THEN lookup (if valid)
- Submit: `validateAll()` runs on submit, blocks if any error

### Dark Mode (existing requirement — must not regress)

All new CSS uses only semantic tokens (`--color-surface`, `--color-surface-muted`, `--color-border`, `--color-text-*`, `--color-primary`). No raw hex values. No `--color-gray-*` primitive tokens directly in component CSS. Dark mode coverage is automatic via existing `[data-app-theme="dark"]` overrides in tokens.css.

## Accessibility Contract

| Requirement | Contract |
|-------------|----------|
| WCAG 2.1 AA contrast (A11Y-01) | All new text uses semantic tokens with established contrast; metadata label `--color-text-muted` on `--color-surface-muted` must be verified — target 4.5:1 |
| Semantic HTML (A11Y-02) | Age display uses `<p>` with `aria-live="polite"`; metadata group uses `<section aria-label="Dados Administrativos">`; CPF spinner uses `<span aria-label="Buscando paciente...">` |
| Focus indicators (A11Y-03) | All inputs inherit existing focus ring pattern (teal border + 3px box-shadow); no new focus pattern introduced |
| Form labels (FORM-01) | All inputs (primary + metadata) have associated `<label>` via `htmlFor` + matching `id`; no label-less inputs |
| aria-invalid on error | `aria-invalid="true"` on `.patient-form__input--error` inputs (existing pattern — keep) |
| aria-describedby for errors | `aria-describedby="{field}-error"` wired to `.patient-form__error` element (existing pattern — keep) |

**Contrast note on metadata section label:** `--color-text-muted` on `--color-surface-muted` in light mode resolves to `--color-gray-400` (#a8a29e) on `--color-gray-100` (#f5f5f4). Calculated ratio: ~2.5:1. This does NOT pass 4.5:1 AA for body text. However, the section label is decorative/organizational (not conveying required information) and is supplemented by visual grouping. If the checker flags this, fall back to `--color-text-secondary` (#57534e on #f5f5f4 = ~5.2:1) for the section label text.
