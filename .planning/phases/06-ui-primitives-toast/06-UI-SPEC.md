## Spacing Scale

All values map to existing `--spacing-*` tokens in `src/styles/tokens.css`.

| Token | CSS Variable | px Equivalent | Phase 6 Usage |
|-------|-------------|---------------|---------------|
| xs | `--spacing-xs` | 4px | Icon-to-text gaps, toast close button tap area |
| sm | `--spacing-sm` | 8px | Toast item gap in stack, button internal padding (sm size) |
| md | `--spacing-md` | 16px | Default button padding, toast horizontal padding, toaster right/top offset |
| lg | `--spacing-lg` | 24px | Toast vertical padding, button group gaps |
| xl | `--spacing-xl` | 32px | Section-level separation (not primary in this phase) |
| 2xl | `--spacing-2xl` | 48px | Not used in Phase 6 |

**Exceptions:**
- Toast container `top` offset: `calc(64px + var(--spacing-md))` — 64px is the fixed Header height from `src/components/Header.css`. This is a named exception, not a new token.
- Button minimum height: 36px (`sm`) and 40px (`md`) — ensures WCAG touch target size without a new token.

## Color

| Role | Semantic Token | Light Value | Dark Value | Usage |
|------|---------------|-------------|------------|-------|

| Secondary surface (30%) | `--color-surface-muted` | `--color-gray-100` (#f5f5f4) | `--color-gray-700` (#44403c) | Button secondary background on hover |

### Toast System (`src/components/ui/Toast.jsx` + `Toast.css` + `ToastProvider.jsx`)

| Property | Value | Source |
|----------|-------|--------|
| Position | `fixed`, top-right | CONTEXT.md D-02 |

| Right offset | `var(--spacing-md)` | RESEARCH.md |

| Max width | 360px | RESEARCH.md Pattern |
| Stack limit | 3 toasts; oldest dropped when 4th arrives | CONTEXT.md D-04 |
| Auto-dismiss | 8000ms minimum | CONTEXT.md D-03 / REQUIREMENTS.md INTR-01 |
| Manual close | X button always visible; `aria-label="Fechar notificação"` | CONTEXT.md D-03 |
| Toast types | `error`, `success`, `info` | CONTEXT.md D-05 |
| Border radius | 8px | default; matches `index.css` base button radius |
| Padding | `var(--spacing-md) var(--spacing-lg)` (16px 24px) | default |
| Border | 1px solid — left border 4px solid (type accent) | design default |

**ARIA contract:**

| Type | role | aria-live |
|------|------|-----------|
| error | alert | assertive |
| success | status | polite |
| info | status | polite |

**Critical:** The `Toaster` container div must be rendered unconditionally in the DOM (even when empty) with `aria-live` on it. This ensures screen readers pre-register the live region before toasts appear (NVDA/JAWS compatibility).

## Animation Contract

CSS-only (`@keyframes`) — no animation library. All durations under 300ms.

| Animation | Duration | Easing | Trigger | Definition |
|-----------|----------|--------|---------|------------|
| Toast entrance | 250ms | `ease-out` | Toast mounts | `@keyframes toast-enter`: translateX(100%)→0 + opacity 0→1 in `Toast.css` |
| Toast exit (exiting flag) | 200ms | `ease-in` | Dismiss called | `@keyframes toast-exit`: opacity 1→0 in `Toast.css`; requires `exiting` flag + `setTimeout(removeToast, 200)` |
| Button spinner | 600ms | `linear` (infinite) | `loading` prop true | `@keyframes btn-spin` in `Button.css`; `::after` pseudo-element border-spin |
| Chat bubble appear | 200ms | `ease-out` | Message renders | Reuse existing `@keyframes fadeIn` from `index.css` via `.animate-fade-in` utility class |

**`.animate-fade-in` utility class** — add to `src/index.css` after existing `@keyframes fadeIn`:

```css
.animate-fade-in {
  animation: fadeIn 200ms ease-out forwards;
}
```

Apply to chat bubble `<div key={idx}>` wrapper in `ProtocolTriage.jsx` — add `className="animate-fade-in"` alongside existing inline styles. No structural changes.

**Note on translateY in chat bubble animation:** The existing `fadeIn` uses `translateY(5px)→0`. Add `will-change: opacity, transform` to `.animate-fade-in` to promote to compositor layer and prevent layout jitter with auto-scroll (`scrollIntoView`).

```

    <UserProvider>
      <AppContent>
```

`ToastProvider` must be OUTSIDE `BrowserRouter` so the `Toaster` (fixed-position) is not remounted on route changes.
