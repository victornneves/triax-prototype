<objective>
Enrich the history list so clinicians can identify past sessions at a glance via MTS priority badge, patient name, and session duration. Migrate HistoryPage from S3-legacy patterns (item.key, item.lastModified, ?key= query) to proper session-id-based fields from the DynamoDB-backed API.

Purpose: The current history list shows only date, time, and a truncated S3 key — meaningless to clinicians. The API already returns rich session data (patient_info, triage_result, stats) that HistoryPage ignores. This plan wires the existing data into a clinically useful table.

Output: `src/utils/priority.js` (shared utility), updated `Profile.jsx` (import migration), updated `HistoryPage.jsx` + `HistoryPage.css` (enriched 4-column table with priority badge, patient name, duration, and session-id-based detail fetch).
</objective>

    ```javascript
        const p = (
            triageResult?.prioridade ||
            triageResult?.cor ||
            triageResult?.priority ||
            ''
        ).toLowerCase();

        if (p.includes('red') || p.includes('vermelho'))    return { ...PRIORITY_MAP.red };
        if (p.includes('orange') || p.includes('laranja'))  return { ...PRIORITY_MAP.orange };
        if (p.includes('yellow') || p.includes('amarelo'))  return { ...PRIORITY_MAP.yellow };
        if (p.includes('green') || p.includes('verde'))     return { ...PRIORITY_MAP.green };
        if (p.includes('blue') || p.includes('azul'))       return { ...PRIORITY_MAP.blue };
        if (PRIORITY_MAP[p]) return { ...PRIORITY_MAP[p] };
        return {
            className: 'priority-badge priority-unknown',
            label: triageResult?.prioridade || p || 'N/A',
        };
    }
    ```

    3. `formatDuration(seconds)` function (per D-07) — new utility for duration display:
    ```javascript
    export function formatDuration(seconds) {
        if (seconds == null) return '\u2014';
        const mins = Math.floor(seconds / 60);
        const secs = seconds % 60;
        if (mins === 0) return `${secs}s`;
        return `${mins}min ${secs}s`;
    }
    ```
    The `\u2014` is an em dash character. `seconds == null` covers both null and undefined.
